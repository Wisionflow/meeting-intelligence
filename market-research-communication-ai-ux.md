# Исследование рынка: UI/UX для Communication Intelligence

**Дата:** 22.03.2026  
**Продукт:** Communication Intelligence — стратегический AI-ассистент для деловой коммуникации

---

## Раздел 1: Топ-5 продуктов для изучения

### 1. Crystal Knows — crystalknows.com
**Что делает:** DISC-based personality advisor. Показывает профиль человека по LinkedIn, даёт советы «как писать этому человеку», готовит pre-meeting playbooks.

**Интерфейс:**
- Chrome extension — sidebar поверх LinkedIn/Gmail
- На странице контакта: DISC-тип + краткая характеристика + 5-10 советов по коммуникации
- Структура карточки: большой заголовок типа (D/I/S/C) → 3 блока: как общаться / чего избегать / как мотивировать
- В Gmail-режиме: под полем ввода письма — предложения как адаптировать тон

**Что взять:**
- Карточка профиля получателя как постоянный элемент боковой панели
- Формат «do / don't» для конкретного профиля — визуально очень понятен нетехнику
- Pre-meeting playbook как шаблон перед встречей

**Чего нет:**
- Не анализирует ситуацию, только профиль
- Не учитывает профиль отправителя
- Не пишет письмо — только советует
- Нет объяснения «почему именно так»

---

### 2. Lavender — lavender.ai
**Что делает:** Real-time email coach. Анализирует письмо пока пишешь, ставит оценку 0-100, объясняет что плохо и почему, предлагает исправления.

**Интерфейс:**
- Sidebar в Gmail/Outlook (правая панель, ~350px)
- Верхний элемент: большой score-кружок (0-100) — моментально обновляется при изменении текста
- Под скором: три секции — "Необходимые исправления", "Предложения", "Что хорошо"
- Каждый пункт: иконка-маркер + текст + кнопка "вставить исправление"
- Отдельная вкладка "Prospect" — данные о получателе
- Mobile preview — как письмо выглядит на телефоне

**Ключевой UX-паттерн:**  
Coaching с объяснением «почему» = то, что Lavender делает лучше всех. Пример: не «укороти предложение», а «предложения длиннее 20 слов снижают reply rate на 18%».

**Что взять:**
- Score как главный элемент — создаёт геймификацию и мгновенный feedback
- Объяснение «почему» после каждого совета — это наш ключевой дифференциатор
- Трёхуровневая структура ответа (критично / желательно / хорошо)
- Кнопка «Regenerate» для альтернативных вариантов каждого предложения

---

### 3. Superhuman AI + Shortwave
**Что делают:** AI-native email clients — AI встроен в сам интерфейс, а не как addon.

**Ключевые паттерны:**
- «AI summaries» поверх длинных тредов — сворачиваемый блок в верхней части переписки
- «Reply suggestions» — 2-3 варианта ответа одним кликом, можно редактировать inline
- «Ask AI» — диалоговое окно в контексте конкретного письма
- Keyboard shortcuts для всех AI-действий (снижает барьер входа для power users)

**Что взять:**
- Inline editing прямо в AI-блоке — золотой стандарт для профессионалов
- Reply suggestions как first-click UX — показать 2-3 варианта стратегии до диалога
- «Ask AI about this thread» — шаблон для «добавить контекст» без перезапуска

---

### 4. Perplexity / Claude — structured AI chat
**Релевантность:** Лучшие примеры структурированных ответов в чат-интерфейсе.

**Ключевые паттерны:**
- Collapsible «Thinking...» блок (Chain of Thought) — accordion с chevron, свёрнут по умолчанию
- Markdown-рендеринг в пузырях — заголовки, списки, bold
- Copy-кнопка на каждом блоке/секции
- Citations как inline-ссылки
- «Follow-up questions» предложения внизу ответа

**Данные по CoT display (исследование digestibleux.com 2025):**
- ChatGPT: свёрнутый accordion, короткий label — наименьшая когнитивная нагрузка ✅
- Claude: структурированный, с bullets — читабельно
- DeepSeek: всё открыто, вертикальный scroll — overwhelming для нетехника ❌
- Вывод: **свёрнутый accordion с "Анализирую ситуацию..."** — оптимально для нашего пользователя

### 5. ShapeOfAI.com — паттерн-библиотека AI UX
**Не продукт, а исследовательский ресурс** с каталогом UX-паттернов для AI.

**Самые релевантные паттерны для нас:**
- **Stream of Thought** — bounded box с reasoning, свёрнут по умолчанию, раскрывается по клику
- **Structured Templates** — шаблоны ввода которые можно заполнить (вместо пустого поля)
- **Prompt Suggestions** — стартовые подсказки снимают blank canvas anxiety
- **Feedback Mechanisms** — thumbs up/down на уровне каждого блока

## Раздел 3: Антипаттерны — что точно НЕ делать

### 1. Длинный CoT всегда открыт
Пользователь приходит за ответом, не за рассуждением. DeepSeek-стиль (всё открыто, бесконечный scroll) — 100% killing feature для занятого директора. Reasoning должен быть доступен, но свёрнут.

### 2. Один большой текстовый блок
Если 5-блочный анализ выходит одной стеной текста — пользователь читает первые 2 абзаца и уходит. Структурные заголовки + accordion — обязательно.

### 3. Кнопка «отправить» встроена в AI-ответ
Создаёт ощущение что AI отправит сам. Паранойя у корпоративных пользователей. Всегда: сначала «скопировать» / «редактировать», потом пользователь сам отправляет.

### 4. Onboarding через длинную форму заполнения профилей
Crystal Knows делает правильно: DISC-тест 10 минут, и ты уже в системе. Если заставить вводить профили вручную — 80% уйдут на шаге 1.

### 5. Нет «Plan B»
Пользователь всегда думает «а что если не сработает». Если в ответе нет запасного варианта — доверие падает. Даже короткий блок «если не ответит за 3 дня...» критически важен.

### 6. Generic тон-совет без привязки к профилю
«Пишите более официально» — бесполезно. «Мельников — D-тип, ему нужна цифра в первом предложении» — ценно. Без профильной привязки это просто Grammarly.

### 7. Мобильный = просто уменьшенная десктопная версия
5 блоков полного анализа на мобильном — нечитабельно. Мобильный режим должен показывать: (1) готовый текст + (2) одну ключевую стратегическую подсказку. Остальные блоки — по клику.

### 8. Нет состояния «думает»
Если запрос идёт 5-15 секунд без индикатора — пользователь нажимает кнопку снова. Streaming ответа (токен за токеном) или как минимум анимация «Анализирую профили...» → «Формирую стратегию...».

## Раздел 4: Быстрые победы (1-2 дня в communicate.html)

### ПБ-1: Collapsible analysis blocks (4-6 часов)
Каждый из 5 блоков — accordion с заголовком и chevron. По умолчанию блок 3 (текст письма) открыт, остальные свёрнуты. Пользователь разворачивает нужное.

**Эффект:** Страница не пугает объёмом. Директор сразу видит готовый текст.

### ПБ-2: Copy-кнопка на каждый блок (2 часа)
Под каждым accordion-блоком — кнопка «📋 Скопировать». При нажатии — 2 секунды «Скопировано ✓».

**Эффект:** Первое, что спросит пользователь — «как скопировать только письмо». Снимаем вопрос до того как он возник.

### ПБ-3: Recipient mini-card в header (3-4 часа)
Над полем ввода — маленькая карточка: фото/инициалы + DISC-тип + 2-3 ключевые черты. Фиксированная, всегда видна.

**Эффект:** Пользователь помнит кому пишет, даже когда листает длинный ответ.

### ПБ-4: Tone quick-switch (2-3 часа)
Под блоком с текстом письма — 4 кнопки: [Мягче] [Жёстче] [Формальнее] [Короче]. При нажатии — только этот блок перегенерируется, остальные не трогаются.

**Эффект:** Самый частый follow-up запрос пользователя — «а теперь напиши это немного мягче». Делаем одним кликом вместо нового промпта.

### ПБ-5: «Thinking» streaming indicator (1-2 часа)
Вместо пустого экрана пока идёт запрос — последовательные сообщения:
- «🔍 Анализирую профили участников...»
- «⚡ Определяю динамику взаимодействия...»
- «✍️ Формирую стратегию и текст...»

**Эффект:** Снимает тревогу ожидания. Показывает что система «думает», а не зависла.

## Раздел 5: Российский рынок — выводы

**Прямых конкурентов нет.** Проведённый анализ показывает:

- **Email AI (App Store)** — только генерация писем, без профилей, без стратегии
- **ГигаЧат / YandexGPT** — general purpose, нет специализации на деловых конфликтах
- **Mymeet.ai** — транскрибация совещаний, не коммуникационная стратегия
- **Битрикс24 AI** — встроен в CRM, фокус на задачах, не на межличностной динамике

**Вывод:** Ниша «стратегический советник по деловым коммуникациям с учётом психологических профилей» в России не занята. Ближайший западный аналог — Crystal Knows + Lavender в одном продукте, но адаптированный под российскую корпоративную культуру (директивность, иерархия, «не выносить сор из избы»).

**Что важно для российского B2B пользователя:**
1. Данные не уходят за рубеж — можно позиционировать как локальное решение
2. Понимает русскую корпоративную культуру (DISC D-типы в топ-менеджменте, культура замалчивания конфликта)
3. Интеграция с Битрикс24 / Яндекс-экосистемой — быстрый вход без нового login
